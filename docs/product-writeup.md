# 产品说明(≤500 字)

**怎么定义"一份好的 PRD 大纲"?**
五条:① 结构完整(背景/用户/目标/需求/边界全覆盖);② 对不确定性诚实——区分事实与假设,绝不编造人名数字;③ 决策导向——有「非目标」与优先级,讲清不做什么;④ 具体可执行;⑤ 能迭代的活文档。工具每个功能都对应其中一条。

**做了哪些关键产品决策?**
1. "协作者"交互:先出草稿再精修,把"信息不足怎么办"从缺陷变成卖点——缺口显式变成假设与澄清问题。
2. 脚手架 + 留白:给完整框架,只填有把握推断的内容,其余细节(含未知事实)留 `[待填写]` 交给用户——不编造、用户掌控、输出克制。
3. 版本号与时间由应用确定性管理,不交给模型。
4. 无 key 自动进 Demo 模式,任何人打开都能体验完整流程。
5. 双轨编辑:对话管"整体 / 答疑",划选 inline 管"改具体一句"——避免小改动触发整篇重写。

**再给一周会优化什么?**
支持上传文件(图片 / PDF / 文档)作为参考来源,让生成更贴合真实材料;导出 PDF/Word、推送 Notion/飞书;按产品类型自适应模块;收窄对话职责 + 撤销/重做;PRD 质量自评分;多语言。
